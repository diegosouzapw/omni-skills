# TASK-03 Positioning And Documentation Refresh

Read this file before starting implementation.

## Goal

Make the project description consistent everywhere.

## Required Positioning

The docs must clearly state that the repository is:

1. a public repository of skills
2. a repository of curated improved skills following best practices
3. a runtime surface for the same catalog

## Scope

- root README hero and opening sections
- docs index
- contributing guide
- skill PR workflow
- npm description text
- repository description text

## Deliverables

- revised positioning copy
- clearer explanation of `skills/` vs `skills_omni/`
- explicit mention of direct PR intake and repository-based external intake

## Acceptance

- a new reader can understand the native and curated surfaces within the first screen of the README
- no key top-level doc still frames the project as only a runtime or only a markdown repository
