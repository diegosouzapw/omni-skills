# Design Token Governance Checklist

- Confirm the distinction between primitive, semantic, and theme override tokens.
- Identify which token names are public contract and which remain implementation detail.
- Review aliases, fallback rules, and deprecation windows before removal.
- List downstream consumers: components, apps, docs, themes, Figma variables.
- Define verification scope for light, dark, branded, and accessibility-sensitive themes.
- Publish owner, migration path, and rollout timing with the token change.
