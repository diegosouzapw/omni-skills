# Security Audit Checklist

## Scope

- Define the attack surface, trust boundaries, and assets under review.
- Separate policy non-compliance from exploitable risk.
- Prioritize by impact and likelihood, not only by category.

## Findings

- Each finding should include evidence, exploit path, and remediation guidance.
- Prefer precise, reproducible statements over broad fear-based language.
- Distinguish confirmed issues from suspicious signals needing follow-up.

## Delivery

- End with severity-ranked findings and a shortest-safe remediation order.
- Call out verification gaps and assumptions.
- Keep the report useful for both engineers and reviewers.
