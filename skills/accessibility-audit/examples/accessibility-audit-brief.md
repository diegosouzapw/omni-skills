# Accessibility Audit Brief

## Scope

- screen: settings modal
- devices: desktop and mobile web
- focus areas: keyboard path, validation errors, reduced motion, focus visibility

## Release blockers

- first focus lands on a non-interactive wrapper
- validation errors are visual only
- reduced-motion path still uses animated panel sweep

## Follow-ups

- clarify heading structure
- improve toggle naming consistency
- strengthen focus contrast on dark surfaces
