# Frontend Design Checklist

## Product Intent

- Clarify the primary user task before choosing layout or motion.
- Define the critical states: loading, empty, error, success, and responsive variants.
- Preserve the established design system when the repo already has one.

## Visual Quality

- Use a deliberate hierarchy for typography, spacing, and contrast.
- Avoid generic default layouts when the task calls for a stronger concept.
- Keep accessibility and touch targets in scope while shaping the UI.

## Delivery

- End with concrete component structure and interaction notes.
- Verify responsiveness on narrow and wide viewports.
- List any intentional deviations from mocks or existing patterns.
