# Usage, Billing, and Retention Guardrails

Use these guardrails when expansion depends on consumption, overages, or usage thresholds.

## Core principles

- customers should understand what drives cost
- alerts should arrive before surprise invoices
- invoice language should be plain enough for non-experts
- strategic accounts need a human escalation path
- payment recovery should be separated from product-fit analysis

## Trigger design checklist

For each threshold, define:

- threshold level
- customer-facing message
- owner
- customer safeguard
- fallback path

## Recommended customer safeguards

- 50% usage alert with visibility only
- 75% alert with tier comparison and forecasted implications
- 90% alert with clear next-step options
- spend caps or approval path where supported operationally
- temporary grace path for strategic accounts when appropriate

## Risks to avoid

- surprise overages
- aggressive prompts without value context
- unclear invoice lines
- auto-upgrade recommendations without commercial transparency
- treating payment failure as proof of low product value

## Operational note

If support complaints or invoice disputes rise after expansion changes, pause optimization and re-check customer trust signals before scaling the motion.
