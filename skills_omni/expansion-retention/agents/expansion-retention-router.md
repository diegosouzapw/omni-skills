# Expansion & Retention Router

Use this note when the request starts in retention strategy but drifts into adjacent work.

## Stay in this skill when

- the user needs churn diagnosis
- the user needs a health-score framework
- the user needs activation/onboarding operating design
- the user needs expansion trigger design
- the user needs renewal-risk or win-back strategy
- the user needs owner maps, workflows, and playbooks

## Hand off when

### Pricing redesign becomes primary

Route to a pricing skill when the user needs:
- packaging redesign
- tier architecture
- monetization strategy overhaul
- discount policy design
- pricing research or willingness-to-pay work

### Analytics instrumentation becomes primary

Route to analytics / data skill when the user needs:
- event taxonomy
- warehouse modeling
- dashboard queries
- retention or activation instrumentation implementation
- product analytics platform setup

### CRM / RevOps build-out becomes primary

Route to revops / implementation skill when the user needs:
- HubSpot or Salesforce workflows
- lifecycle automation setup
- routing rules
- object model changes
- playbook implementation inside a platform

### Copywriting becomes primary

Route to messaging / outreach skill when the user needs:
- lifecycle email copy
- win-back sequence copy
- in-product upgrade messaging
- call scripts and objection handling assets

### Compliance interpretation becomes primary

Route to legal/compliance review when the recommendation depends on:
- consent interpretation
- privacy law interpretation
- regulated-industry outreach restrictions
- approved data-use boundaries

## Handoff prompt examples

### To analytics skill

```text
Take this retention strategy and translate it into an event instrumentation plan for activation, adoption breadth, and renewal-risk dashboards.
```

### To CRM / RevOps skill

```text
Implement these triggers, owners, and SLAs in our CRM and CS tooling. Preserve the operating logic from @expansion-retention.
```

### To pricing skill

```text
The retention issue appears driven by pricing and packaging mismatch. Reframe this as a pricing redesign problem rather than a CS workflow problem.
```
