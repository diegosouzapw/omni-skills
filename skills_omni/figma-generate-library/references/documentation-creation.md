# Documentation and File Structure Reference

Use this reference in Phase 2.

## Goals

- make the file navigable
- document foundations before large-scale component creation
- prepare reviewers and developers for later handoff

## Recommended page skeleton

- Cover
- Getting Started
- Foundations
- ---
- Components
- ---
- Utilities

Adapt to existing conventions if the file already has a strong structure.

## Foundations documentation ideas

Include concise documentation for:
- color tokens and theme behavior
- typography scales
- spacing scale
- radius scale
- elevation/effect styles if used

The goal is clarity, not decorative polish.

## Component page guidance

For each component page, include:
- title
- brief description
- variant/property summary
- simple usage guidance
- states or edge cases if important

## Review artifact guidance

Before leaving Phase 2, capture:
- page list
- screenshot of the page structure or foundations overview
- notable documentation gaps that will be filled later

## Common mistakes

- skipping foundations docs entirely
- mixing many unrelated components on one page by default
- creating a file structure that does not match the intended system scale
