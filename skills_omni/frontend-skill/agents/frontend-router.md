# Frontend Router

Use this note when `frontend-skill` is no longer the best primary skill.

## Stay in `frontend-skill` when

- the core task is page composition
- the work is mostly responsive layout, hierarchy, imagery, motion, and UI shell implementation
- accessibility and performance checks are frontend-facing and local to the page or surface

## Hand off to another skill when

### `@chatgpt-apps`

Use when the task becomes Apps SDK integration, tool descriptors, component wiring inside ChatGPT, or app-runtime behavior.

### `@aspnet-core`

Use when API shape, backend endpoints, server logic, authentication, or backend rendering concerns dominate the task.

### `@develop-web-game`

Use when gameplay systems, engine logic, scene management, or advanced game interactions become primary.

### `@doc`

Use when the primary deliverable is documentation, structured handoff writing, or broader product documentation rather than frontend implementation.

## Handoff rule

When handing off, pass along:

- deliverable type
- visual thesis
- current layout or surface structure
- accessibility concerns already identified
- performance concerns already identified
- any preserved upstream provenance notes
