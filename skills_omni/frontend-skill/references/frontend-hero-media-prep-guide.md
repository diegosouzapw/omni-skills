# Frontend Hero Media Prep Guide

Hero media often decides whether the page feels premium or merely busy.

## Choose the right asset

- Use imagery that does narrative work.
- Prefer real-looking, in-situ photography or product visuals over abstract filler.
- Avoid images with embedded UI frames, signage, competing logos, or typographic clutter.

## Plan for text placement

- Choose a stable tonal area for the text column.
- Keep the text column narrow enough to scan quickly.
- Do not depend on excessive overlays to rescue a bad image.
- If text cannot remain legible across likely crops, choose a different asset.

## Crop and art direction

- Define the focal point explicitly.
- Check desktop, tablet, and mobile crops separately.
- Use alternate crops when one master image fails on small screens.
- If multiple moments are needed, use multiple images rather than one collage.

## Responsive delivery

- Use responsive images when different sizes or crops materially improve delivery.
- Do not send oversized desktop imagery to mobile by default.
- Lazy-load non-critical media below the fold.

## Accessibility

- Informative hero imagery should have meaningful alternative treatment.
- Decorative hero imagery should not create noise for assistive technology.
- Confirm text over imagery remains readable and interactive controls remain clear.

## Performance prompts

- Is the hero likely the LCP element?
- Can the file be smaller without harming the visual thesis?
- Are media dimensions reserved to reduce layout shift?
- Is a static image enough where video was being considered?
