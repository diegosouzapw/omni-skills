# Frontend Handoff Checklist

Use this before merge, review, or delivery.

## What to include

- short statement of the deliverable type
- visual thesis
- section sequence or app structure
- primary CTA or primary user action
- media decisions that materially affect layout
- motion decisions and reduced-motion behavior

## Evidence to provide

- screenshots or notes for mobile and desktop first viewport
- confirmation of keyboard and focus review
- note on text-over-image contrast review if applicable
- note on likely LCP element and any major performance risks
- note on layout-shift protections for media or embeds

## Handoff questions

- What should the reviewer verify manually?
- Which areas are intentionally restrained versus unfinished?
- Which motion patterns are required versus optional polish?
- Is any related skill handoff still needed?

## Final litmus checks

- Is the brand or product unmistakable in the first screen?
- Does each section have one job?
- Are cards actually necessary?
- Is motion improving the result rather than decorating it?
- Would the page still feel premium if shadows and extra chrome were removed?
