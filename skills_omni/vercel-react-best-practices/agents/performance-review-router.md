# Performance Review Router

Use this router when performance work crosses into another specialization.

## Stay in this skill when

- the core problem is React rendering behavior
- the core problem is Next.js App Router architecture
- the main change involves data fetching, caching, bundle size, hydration, images, fonts, or scripts

## Hand off when

### To deployment or platform skills

- rollout behavior differs by environment
- CDN, runtime, or hosting configuration is the main bottleneck
- the issue is more about build output or deployment topology than component architecture

### To security-focused skills

- the review becomes primarily about auth, authorization, validation, CSP, or secret handling
- a performance optimization introduced sensitive exposure risk

### To observability or diagnostics skills

- the team needs deeper production measurement, tracing, or telemetry design
- the bottleneck cannot be localized from route-level review alone

### To accessibility or design skills

- a performance fix changes UX semantics, usability, or accessibility behavior materially
