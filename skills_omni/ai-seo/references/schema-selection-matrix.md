# Schema Selection Matrix

Use schema only when it matches visible content on the page.

## Matrix

| Page type | Possible schema | Use only if | Avoid when |
| --- | --- | --- | --- |
| Competitor alternative page | FAQPage | the FAQ questions and answers are visible on-page | the FAQ is hidden, thin, or not user-facing |
| Product or solution page | Product | the page clearly describes a product and relevant visible properties | the page is only a broad category page |
| Editorial review or comparison page | Review | there is a real editorial review visible on the page | there is no actual review content |
| How-to guide | HowTo | the page contains a true step-by-step process | the page is mostly promotional or comparative |

## Validation rules

- Markup must reflect visible content.
- Do not add schema just because a template allows it.
- Valid schema does not guarantee a rich result.
- Re-check eligibility after template changes.

## Operator checklist

- [ ] Does the page visibly contain the marked-up content?
- [ ] Is the schema type appropriate to the page?
- [ ] Would a user see the same information without reading the code?
- [ ] Has the page been rechecked after template edits?
