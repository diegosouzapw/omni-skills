# SEO Intake Decision Tree

Use this decision aid before recommending net-new content.

## 1. Check existing evidence first

Ask:

- Does the site already have a page on this topic?
- Does Google Search Console show impressions for related queries?
- Is CTR weak, or is the page absent entirely?

If **yes**, start with **optimize existing content** before creating a new page.

## 2. Choose the motion

### Optimize Existing Page

Choose this if:

- relevant impressions already exist
- the page ranks but does not earn clicks
- intent is partially matched but execution is weak
- the page can be expanded, clarified, or refreshed

### Competitor Alternative Page

Choose this if:

- buyers compare you against named competitors
- commercial intent is clear
- you can support claims with current evidence
- comparative review can be maintained over time

### Comparison Page

Choose this if:

- the query is explicit product-vs-product evaluation
- a balanced side-by-side format is possible
- pricing/features/use-case differences can be cited

### Programmatic Cluster

Choose this if:

- the topic is a repeatable entity pattern
- each page can include unique fields or logic
- the team can test a pilot cohort first
- internal linking and sitemap coverage can be managed

## 3. Reject the motion if any of these are true

Do not scale yet if:

- pages would mostly be token swaps
- no evidence owner exists
- there is no reliable source data
- freshness cannot be maintained
- the domain has no realistic path to compete on the selected pattern

## 4. Required intake fields

Record:

- business goal
- chosen motion
- evidence source
- existing pages checked
- proprietary data available
- review owner
- legal review needed
- success metric
