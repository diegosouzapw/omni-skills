# Session Handoff Checklist

Use this before pausing work and again when resuming.

## Before pausing

- Update `STATE.md`
- Record completed work
- Record blockers and open questions
- Record assumptions made during execution
- Record deferred ideas that should not be forgotten
- Record the exact next resume point
- Note any unverified changes or failed checks
- Note any repo-state concerns, branch context, or local changes if relevant

## `STATE.md` recommended sections

- Decisions
- Open Questions
- Blockers
- Assumptions
- Deferred
- Next Resume Point

## When resuming

- Read `STATE.md` first
- Confirm the repository still matches the recorded state
- Reopen only the active feature packet
- Reconcile any drift before making new edits
- Update the state if the plan changed during reconciliation
