# Value-Before-Purchase Selector

Pick the proof mechanism based on what the buyer needs to believe.

## If the buyer needs to see the workflow

Use:

- interactive demo
- guided walkthrough

Best for:

- complex UI products
- products with visible workflow benefits

## If the buyer needs to test with realistic data or environment

Use:

- sandbox
- sample workspace
- guided proof of concept

Best for:

- developer tools
- analytics products
- integration-heavy software

## If the buyer needs evidence of outcome on their current state

Use:

- audit
- assessment
- scan
- benchmark report

Best for:

- security
- optimization
- infrastructure analysis

## If the buyer needs to estimate business impact

Use:

- ROI calculator
- cost model
- business case worksheet

Best for:

- high-ACV or CFO-visible purchases

## If the buyer needs to understand how to get started

Use:

- prebuilt template
- starter workflow
- sample project

Best for:

- workflow tools
- collaboration software
- automation products

## If the buyer needs stakeholder alignment

Use:

- workshop
- tailored demo session
- scoped POC

Best for:

- enterprise, multi-stakeholder deals

## Output

Recommend:

1. proof mechanism
2. target segment
3. why this mechanism fits
4. what success signal it should generate
