# PQL Design Worksheet

Use this when the recommended motion includes product-to-sales handoff.

## 1. Fit signals

Examples:

- company size in target range
- target industry or use case
- role or team matches ICP
- domain quality or account type

## 2. Intent signals

Examples:

- pricing page visits
- admin or billing exploration
- security or SSO documentation views
- request for enterprise features
- repeat team invites

## 3. Value signals

Examples:

- activation event completed
- usage crosses a meaningful threshold
- second workflow, integration, or project launched
- multiple active users from one account
- production or business-critical usage begins

## 4. Exclusion rules

Examples:

- student or hobby usage
- free users below ICP
- bot, abuse, or spam patterns
- one-time activity without repeat usage

## 5. Qualification window

- What time period matters: 7, 14, or 30 days?
- Which signals must occur within that period?

## 6. Handoff rule

A PQL fires only when:

- fit is present
- value is demonstrated
- intent or expansion signal appears
- exclusion rules do not apply

## 7. Sales action after PQL

Define exactly what sales should do differently.

Examples:

- offer rollout help
- discuss security and procurement
- support multi-team expansion
- guide enterprise packaging

## 8. Review questions

- Would a rep be glad to get this account?
- Does this handoff happen after value is visible?
- Is sales adding something the product alone cannot?
