# Incident Response Checklist

## First 10 Minutes

- Confirm incident owner or commander
- State user impact and suspected blast radius
- Confirm severity or temporary severity
- Start an incident timeline
- Set the next update time

## Mitigation Review

- List reversible actions first
- Note dependencies or external providers involved
- Capture rollback and feature-flag options
- Record the operator who can execute each mitigation

## Communication

- Internal responders know the next checkpoint
- Stakeholders have an explicit update cadence
- Facts and hypotheses are separated

## Closeout

- Residual risk is stated explicitly
- Follow-up owner is assigned
- Timeline and action list are saved before closure
