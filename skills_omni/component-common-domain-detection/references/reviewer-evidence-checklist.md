# Reviewer Evidence Checklist

A recommendation should be reviewable without redoing the entire analysis.

- [ ] Scope is stated clearly.
- [ ] Candidate groups are listed with repository paths.
- [ ] Semantic validation notes are present.
- [ ] Bounded-context alignment is stated as yes, unclear, or no.
- [ ] Impact snapshot includes consumers, ownership spread, and release coordination.
- [ ] Recommendation names one primary outcome.
- [ ] Rejected alternatives are documented.
- [ ] Security or operational boundary changes are called out.
- [ ] Confidence and evidence gaps are explicit.
- [ ] Provenance includes paths, files sampled, and commit or PR context when available.
