# Update vs Create Checklist

Use this checklist before writing to Notion.

## Update an existing page if

- an authoritative page already exists for the same topic
- the new material is a correction, status change, or extension
- existing relations, backlinks, or references should remain in one place
- readers would expect one canonical page rather than multiple small pages

## Create a new page if

- the topic is materially new
- the change introduces a separate decision, guide, or reference item
- adding the content to the current page would reduce clarity
- the destination model expects one record per decision, FAQ, or meeting

## Before updating

- search for all plausible existing pages
- fetch the current properties
- read the current body or relevant sections
- note what should be preserved
- decide whether to revise, append, or split content
- add a short what-changed note if appropriate

## Warning signs

Pause and ask the user before proceeding if:

- two candidate pages appear equally authoritative
- the existing page is stale and may no longer be canonical
- the update would replace substantial content without clear justification
- the source material conflicts with the current page
