# Page QA Checklist

Use this before handoff or completion.

## Structure

- title is clear and specific
- first paragraph provides a useful summary
- headings match the page type
- content is scannable and concise

## Metadata

- required properties are populated
- owner and status are set if expected
- tags and relations are appropriate
- date fields are correct

## Provenance

- source links are present
- capture date is recorded if relevant
- unverified claims are marked clearly
- canonical references are linked when available

## Quality

- no obvious duplication with existing pages
- update vs create choice was deliberate
- page can be understood without reading the source thread
- what changed is described for updates

## Safety

- no unnecessary confidential raw transcript was pasted
- open questions remain labeled, not disguised as facts
- links point to the right workspace or canonical sources
