# Notion Schema Checklist

Confirm destination schema details before creating or updating content.

## Core checks

- correct workspace
- correct parent page or database/data source
- required title property
- exact property names and types
- relation targets exist and are accessible
- destination accepts the intended page type

## Common properties to confirm

- Title
- Owner
- Status
- Tags
- Date
- Team
- Project
- Decision type or doc type
- Summary or description
- Review date
- Source link

## Ask before writing

- Is this a standalone page or a database item?
- Which properties are required versus optional?
- Are status values controlled?
- Are relations mandatory?
- Should this page use a template or a specific section order?

## Failure prevention

- do not guess property names
- do not assume text fields exist for provenance
- do not assume successful search means write access exists
- if the schema is unclear, ask the user or inspect the destination first
