# Example: Update Existing Page

## Scenario

The team already has an onboarding page, and a recent discussion changed the recommended setup order.

## Correct workflow

1. Search for the onboarding page.
2. Fetch current page content and properties.
3. Identify which section needs revision.
4. Update only the affected steps and summary.
5. Add a brief what-changed note.
6. Preserve existing links, owners, and related resources.
7. Add the discussion link as provenance for the new guidance.

## Operator note

If the change is so large that the page becomes confusing, ask whether the team wants a new versioned page instead of a direct update.
