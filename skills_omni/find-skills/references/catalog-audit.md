# Catalog Discovery Checklist

## Discovery Rules

- Search the published catalog before inventing a new skill.
- Distinguish installable skills from roadmap-only bundle members.
- Prefer a direct skill match over a broad bundle when the task is already clear.

## Output Quality

- Always echo the exact skill or bundle IDs you are recommending.
- Produce the install command for the user's actual client or target path.
- Say explicitly when the catalog has no published match yet.

## Escalation

- If no skill fits, either solve the task directly or propose creating a new skill.
- If multiple skills fit, explain the tradeoff between narrow and broad installation.
- If the runtime surface is more capable than the catalog, do not overpromise availability.
