# Frontend Blueprint Router

Use this router when the task stops being primarily frontend discovery and design alignment.

## Stay in This Skill When

- visual direction is still unclear
- references need interpretation
- token choices are not yet captured
- implementation order still needs planning
- redesign quality depends on alignment before coding

## Route Out When

### Accessibility-focused work dominates
Route to an accessibility-oriented workflow when the main job is auditing or remediating semantics, keyboard support, contrast, or assistive-technology behavior rather than shaping visual direction.

### Frontend debugging dominates
Route to a debugging workflow when design direction is already approved and the remaining work is fixing implementation defects.

### Design-system governance dominates
Route to a design-system or token-governance workflow when the main task is system definition, naming, migration, or cross-product consistency.

### Review or QA dominates
Route to a UI review or QA workflow when implementation is done and the remaining work is validation before merge.

### Product discovery dominates
Route to product or UX research workflows when the core uncertainty is about user needs or feature definition, not frontend expression.

## Handoff Packet

When routing, pass along:

- approved design direction summary
- token worksheet or equivalent visual decisions
- implementation status
- known accessibility or responsive risks
- unresolved questions
