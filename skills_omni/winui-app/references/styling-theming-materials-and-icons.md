# Styling, Theming, Materials, and Icons

Use this guide when improving visual consistency without fighting the platform.

## Defaults

- Support both light and dark mode unless the user explicitly requires otherwise.
- Prefer theme resources and system brushes over hard-coded colors.
- Keep Fluent-compatible styling and materials aligned with the rest of the app.

## Review questions

- Does the UI remain readable in light and dark themes?
- Are colors and surfaces theme-aware?
- Are icons, typography, and materials consistent with the shell?
- If the title bar is customized, does it still feel integrated and usable?

## Avoid

- hard-coded light-only or dark-only colors
- visual overrides that break contrast
- unnecessary custom surfaces when native styling hooks are enough
