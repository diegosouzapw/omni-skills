## Suggested summary block

```text

- Node: <exact node URL or ID>
- Screenshot: <reference>
- Variables/tokens: <mapping summary>
- Assets: <list>
- Reuse targets: <existing components or primitives>
- Remaining risks: <list>
```

## Handoff rules

- Include the exact node reference, not just a file link.
- Include at least one screenshot reference for parity checking.
- Call out whether the implementation should prioritize an existing design system over literal generated markup.
- Note any localhost or temporary asset URLs that must not ship in final code.
