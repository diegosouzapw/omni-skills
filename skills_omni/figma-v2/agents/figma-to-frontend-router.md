# Figma to Frontend Router

Use this routing note after Figma context has been captured.

## Stay in this skill when

- you still need exact node targeting
- you still need screenshots, variables, or assets from Figma
- you are still resolving MCP setup, access, or node-selection problems
- provenance capture is still incomplete

## Hand off to implementation-oriented skills when

- the exact node and screenshot are already captured
- token or variable mapping notes are already available
- required assets are identified
- the next step is framework-specific coding in the target repository

## Hand off to review-oriented skills when

- implementation exists and the task is now parity review
- accessibility validation is the main remaining concern
- regression testing or visual QA is the main next step

## Minimum handoff packet

Pass along:

- exact Figma file URL
- exact node URL or node ID
- screenshot reference
- variables or token mapping notes
- assets list
- assumptions and unresolved gaps

Use [../examples/figma-handoff-packet.md](../examples/figma-handoff-packet.md) as the standard packet.
