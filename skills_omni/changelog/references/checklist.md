# Changelog Checklist

## Inputs

- Gather merged changes, release notes, migration notes, and user-visible fixes.
- Separate internal refactors from externally relevant behavior changes.
- Confirm the target release tag or date window.

## Structure

- Group entries by Added, Changed, Fixed, Deprecated, or Security.
- Put the user-visible summary first, then deeper implementation notes.
- Call out breaking changes and migration steps clearly.

## Validation

- Remove duplicates and vague wording.
- Verify every high-risk change has a visible explanation.
- Keep the tone factual and scannable for humans and automation.
