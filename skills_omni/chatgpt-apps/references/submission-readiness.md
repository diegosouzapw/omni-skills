# Submission Readiness

Use this checklist only for public app listing or formal review work.

## Prerequisites

- correct organization and role permissions
- production public HTTPS endpoint
- no localhost, private LAN, or tunnel URLs in the final submission path

## Metadata and review materials

Prepare:

- app title and description
- support contact
- privacy policy URL
- required branding assets such as logo or screenshots if requested
- test prompts and expected behaviors
- localization details if relevant

## Auth-specific review readiness

If the app requires login:

- provide review-safe credentials or a reviewer path
- ensure the auth flow works end to end
- avoid requiring private organizational setup that reviewers cannot access

## Common rejection causes

- dev or staging URLs used in submission
- missing support or privacy information
- incomplete metadata
- broken auth review flow
- app behavior not matching the submitted description

## Final check

Before saying an app is submission-ready, confirm that the submission checklist was reviewed separately from local development checks.
