# Creative Testing Playbook

Use this playbook to keep UGC testing interpretable.

## 1. Define the hypothesis

Example: "A demo-first opening will improve early retention versus a spoken pain-point opening for cold TikTok traffic."

## 2. Choose one main variable

Common variables:

- hook
- body structure
- CTA
- creator
- avatar
- offer framing

Change one major variable at a time where possible.

## 3. Hold key conditions stable

Keep stable when possible:

- audience
- offer
- landing page
- placement mix
- budget logic
- attribution window

## 4. Choose metrics by stage

### Leading indicators
- early retention
- hold rate
- CTR
- engagement quality

### Outcome metrics
- CPA
- CVR
- qualified lead rate
- revenue or purchase quality

## 5. Define the decision rule

Write down:

- what counts as enough evidence
- when to kill
- when to refresh
- when to scale
- when to rerun because the test was inconclusive

## 6. Common failure modes

- too many variables changed at once
- budget too small to learn anything useful
- audience changed mid-test
- offer changed while creative was being evaluated
- a permissions or policy issue was mistaken for a creative issue

## 7. Suggested reporting fields

- experiment ID
- objective
- platform
- audience
- variable tested
- constant elements
- spend
- impressions
- clicks
- CTR
- retention notes
- conversion notes
- decision
- next action
