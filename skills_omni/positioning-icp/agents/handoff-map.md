# Handoff Map

Use this map when the work moves beyond positioning, ICP, messaging, or PMF analysis.

## Route to `@ai-pricing` when
- the team needs pricing model design
- willingness-to-pay research is the main task
- packaging and monetization decisions dominate

Input to provide:
- positioning stack
- buyer map
- proof of ROI or time-to-value
- PMF scorecard by segment

## Route to `@ai-sdr` when
- qualification logic is ready
- account prioritization needs sales execution
- sequencing and follow-up operations are next

Input to provide:
- ICP card
- anti-ICP
- fit and intent definitions
- message tiers and proof constraints

## Route to `@ai-cold-outreach` when
- audience, proof, and message are stable
- the next step is writing or testing outbound copy

Input to provide:
- approved value props
- buyer-language notes
- disallowed claims
- regional/privacy caveats if relevant

## Route to SEO or content workflows when
- the team needs pages, articles, or implementation details for comparison content
- publishing and ranking mechanics become the main task

Input to provide:
- comparison brief
- positioning stack
- approved proof claims
- honest gap disclosures

## Route to analytics or product instrumentation workflows when
- PMF measurement requires dashboards, telemetry, or event instrumentation
- retention or cohort analysis cannot proceed without data setup

Input to provide:
- PMF scorecard template
- required metrics and segment cuts
- open data gaps

## Route to legal review when
- the team asks whether enrichment or outreach is lawful in a specific jurisdiction
- claim substantiation or compliance language is disputed

Input to provide:
- intended data sources
- scoring logic summary
- retention and use notes
- exact claims under review
