# Messaging Validation Checklist

Use this checklist before approving product, website, deck, or comparison messaging.

## Core checks

### 1. Specificity
- Does the message name a workflow, pain, or measurable outcome?
- Would a buyer understand what changes after adoption?
- Avoid generic verbs like transform, optimize, streamline without context.

### 2. Differentiation
- Could the top three competitors say the same sentence?
- Is the message tied to a wedge rather than category-level parity?

### 3. Buyer language
- Is the phrasing drawn from customer interviews, sales calls, support tickets, or reviews?
- Does it match the target buyer altitude?

### 4. Proof
- Is there evidence for each strong claim?
- Are date, sample context, and caveats available?

### 5. Altitude match
- Tier 1: strategic narrative for executives
- Tier 2: value propositions for economic buyers
- Tier 3: feature or evaluator detail for practitioners

## AI claim hygiene

Treat these as high-risk claims unless strongly substantiated:
- fully autonomous
- 100% accurate
- hallucinates less than everyone else
- compliant by default
- guaranteed ROI
- replaces entire teams

Safer pattern:
- describe the workflow outcome
- state the conditions
- add the evidence and caveat

## Message test rubric

Score each message 1-5 on:
- comprehension
- relevance
- credibility
- differentiation
- forwarding potential to an economic buyer

A message that scores high on comprehension but low on credibility usually needs better proof, not better writing.
