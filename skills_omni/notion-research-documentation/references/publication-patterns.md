# Publication Patterns

Use these patterns to publish safely into Notion.

## Pattern 1: Draft-first skeleton

Best for long or complex reports.

1. Create the page with:
   - title
   - purpose
   - executive summary placeholder
   - section headings
   - references heading
2. Add content section by section.
3. Review formatting and citations.
4. Mark draft/final status clearly.

## Pattern 2: Append in sections

Best when content is too large or likely to change.

- publish summary first
- append detailed findings next
- append risks, recommendations, and references last

This reduces the chance of brittle large writes.

## Pattern 3: Update with changelog

Best for recurring research outputs.

When updating an existing page:

- preserve the previous structure when possible
- add a short changelog note
- record what changed and why
- keep old references if they still matter

## When to avoid one-shot publishing

Avoid one large create/update when:

- the report is long
- the source set is large
- there are many citations
- the page structure is still changing

Use incremental delivery instead.
