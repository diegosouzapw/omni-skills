# Citation and Provenance Guide

Use consistent citation practice so readers can audit the final document.

## Minimum citation payload for a Notion source

For each cited source, capture:

- page title
- page URL
- page ID if available
- retrieved date
- quoted section or topic area when relevant

## Inline citation guidance

Use inline citations for:

- metrics
- dates
- policy statements
- decisions
- non-obvious factual claims
- controversial or conflicting statements

Example inline style:

```text
The onboarding checklist is maintained separately by each region, creating duplicate process definitions [People Ops onboarding guide, retrieved 2026-03-27].
```

## References section guidance

At the end of the document, include a references section with one entry per source.

Example:

```text
People Ops onboarding guide. Notion page. https://www.notion.so/example Retrieved 2026-03-27.
```

## Evidence vs inference

Keep these distinct:

- **Evidence:** direct source-backed facts
- **Inference:** your synthesis or interpretation
- **Recommendation:** what should happen next

If a recommendation is not explicitly stated in the sources, label it as judgment or recommendation rather than implied fact.

## Contradictions and stale sources

When sources disagree:

- cite both where useful
- note the contradiction explicitly
- prefer newer or more authoritative sources only when you can justify that choice

When a source may be stale:

- include the retrieved date
- mention recency risk in the body or open-questions section
