# Tool Contract Template

Use one compact contract per tool.

## Template

- **Name:**
- **Purpose:**
- **Use when:**
- **Do not use when:**
- **Required inputs:**
- **Optional inputs:**
- **Expected output:**
- **Side effects:**
- **Failure handling:**

## Guidance

- Prefer one short paragraph or bullet list over long prose.
- Remove tools that are irrelevant to the current task.
- Avoid overlapping descriptions that make two tools seem interchangeable when they are not.
- Keep credentials, secrets, and internal-only implementation details out of the prompt.
- If you use MCP tools or resources, treat them as structured context sources, not as justification to paste large tool manuals into the packet.
