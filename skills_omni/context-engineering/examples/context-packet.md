# Context Packet Example

## Section 1: System role

Define the role, safety rules, and non-negotiable policies.

## Section 2: Task state

Include the current user goal, open questions, and relevant working memory.

## Section 3: Retrieved evidence

Provide only the top evidence blocks with source labels and freshness markers.

## Section 4: Tool contracts

List only the tools relevant to this task with preconditions and output expectations.

## Section 5: Omitted context note

State what was intentionally excluded to preserve budget and avoid noise.
