# OpenAI Official Source Map

Use this file as a quick routing layer before searching or fetching docs. It is helper context only. Current official OpenAI documentation remains authoritative.

## Primary official domains

Use only these domains for fallback browsing when MCP is unavailable or unhelpful:

- `developers.openai.com`
- `platform.openai.com`
- `openai.com` for pricing when needed

## Common request types

| Request type | Primary official page | Notes |
| --- | --- | --- |
| General product or API overview | `https://developers.openai.com/` | Good starting point for current docs structure |
| API reference and platform guidance | `https://platform.openai.com/docs` | Canonical API docs hub |
| Quick start | `https://platform.openai.com/docs/quickstart` | Baseline getting-started flow |
| Model choice | `https://platform.openai.com/docs/models` | Verify model capabilities here first |
| Cost-aware selection | `https://openai.com/api/pricing/` | Use alongside model docs |
| Responses API vs Chat Completions | `https://platform.openai.com/docs/guides/responses-vs-chat-completions` | Prefer this for API choice questions |
| Agents | `https://platform.openai.com/docs/guides/agents` | Primary agent orchestration guidance |
| Built-in tools | `https://platform.openai.com/docs/guides/tools` | Tool-using patterns and boundaries |
| MCP | `https://platform.openai.com/docs/guides/mcp` | MCP concepts and setup guidance |
| Prompt guidance | `https://platform.openai.com/docs/guides/prompt-engineering` | Use for prompt-upgrade questions |
| Rate limits | `https://platform.openai.com/docs/guides/rate-limits` | Throttling guidance |
| Error handling | `https://platform.openai.com/docs/guides/error-codes` | Error interpretation and next steps |

## Usage rule

When a bundled local note disagrees with a current official page, prefer the official page and note the discrepancy if it matters to the answer or review packet.
