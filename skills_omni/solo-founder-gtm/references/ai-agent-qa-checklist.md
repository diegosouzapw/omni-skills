# AI Agent QA Checklist

Use this checklist before allowing AI agents into GTM workflows.

## Low-Risk Tasks

These can usually run with sampling and periodic review:

- call summaries
- note cleanup
- categorization and routing
- KPI summaries
- content drafts

## Medium-Risk Tasks

These should draft automatically but require review before customer-facing use:

- outbound personalization
- reply suggestions
- support draft responses
- churn or objection summaries that drive messaging changes

## High-Risk Tasks

These require explicit human approval every time:

- sending outbound messages
- discounts, credits, refunds
- pricing changes
- security, legal, or privacy answers
- claims about integrations, features, or customer outcomes

## Evaluation Questions

- Is the output factually grounded in available inputs?
- Does it avoid invented personalization or invented claims?
- Does it match the founder's tone and positioning?
- Does it stay within compliance and escalation rules?
- Would a founder be comfortable signing their name to it?

## Deployment Rule

Start with one workflow, one rubric, and a small review sample. Only increase autonomy after the error pattern is understood and acceptable.
