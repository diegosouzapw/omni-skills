# Related Skill Router

Use this note to decide whether to stay in skill authoring or hand off.

## Stay in `skill-creator` when

- the main task is creating a new skill
- the main task is improving an existing skill
- the blocker is trigger-writing, structure, support-pack planning, validation, or metadata sync

## Hand off when

### The real work is architecture

Hand off if the user mainly needs system design, tradeoff analysis, or architecture decisions rather than skill packaging.

### The real work is documentation

Hand off if the user mainly needs documentation IA, writing, publishing, or doc maintenance.

### The real work is testing or security

Hand off if the blocker is policy, threat review, static findings, test design, or hardening rather than skill structure.

### The real work is domain execution

Hand off if the user truly wants the domain task completed and the skill itself is already sufficiently authored.

## Handoff packet

When handing off, preserve:

- the target skill name
- the frontmatter trigger boundary
- the current workflow state
- planned or existing support files
- validator status
- any open metadata or drift concerns
