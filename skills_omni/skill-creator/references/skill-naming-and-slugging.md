# Skill Naming and Slugging

Use this guide when naming a new skill or evaluating whether an existing name should change.

## Required rules

- use lowercase letters, digits, and hyphens only
- keep the name under 64 characters
- match the folder name exactly
- avoid spaces, underscores, punctuation, and title case

## Naming priorities

Prefer names that are:

1. specific
2. short
3. descriptive of the workflow
4. likely to help triggering

## Good patterns

- `gh-address-comments`
- `linear-address-issue`
- `docx-redline`
- `pdf-editor`
- `frontend-webapp-builder`

## Weak patterns

- `helper`
- `workflow-tool`
- `misc-docs`
- `project-skill`

## When to include the tool name

Include a tool or platform name when it improves clarity or avoids collisions.

Good:

- `gh-address-comments`
- `linear-address-issue`

Less useful:

- `tool-helper`
- `platform-workflow`

## Rename cautiously

Rename only when the existing name is actively harmful because:

- it is invalid
- it is misleading
- it prevents discoverability
- it conflicts with another skill

If the current name is valid and recognizable, prefer improving the description first.
