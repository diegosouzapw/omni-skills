## 빠른 시작

>**AI 코딩 기술, Claude Code 기술, 커서 기술, Codex CLI 기술, Gemini CLI 기술, 반중력 기술 또는 설치 가능한 `SKILL.md` 라이브러리를 검색하셨나요?**
> 당신은 바로 이곳에 계십니다.### 1️⃣ What is this?

Omni Skills는 AI 코딩 도우미를 위한**설치 가능한 스킬 카탈로그 및 런타임**입니다. 핵심은 재사용 가능한 'SKILL.md' 플레이북의 공개 저장소입니다. 하지만 일반 기술 컬렉션과 달리 저장소는 배포 및 런타임 계층입니다.

## 🧩 Core Concepts

번들을 비교하거나 설치 경로를 선택하기 전에 다음 5가지 구성 요소를 이해하면 도움이 됩니다.

| 개념 | 의미 |
|:---------|:-------------|
| 🧠**기술**| 어시스턴트에게 워크플로를 효과적으로 실행하는 방법을 가르치는 재사용 가능한 `SKILL.md` 플레이북 |
| 📦**카탈로그 아티팩트**| 검색, 비교, 다운로드 및 설치가 가능한 생성된 JSON 및 아카이브 출력 |
| 🔌**MCP 구성**| 보조자가 MCP 도구를 통해 옴니 스킬을 검색할 수 있도록 클라이언트 측 구성 |
| 🤖**A2A 런타임**| 검색, 추천 및 설치 계획 전달을 위한 에이전트 간 오케스트레이션 |
| ✨**선별된 결과물**| `skills_omni/` — 기본 상류 흡입구와 별도로 옴니로 유지 관리되는 향상된 표면 |

>**📝 네이티브/선별 정책:**
> - `skills/`는 모든 언어의 기본 업스트림 섭취를 허용합니다.
> - `skills_omni/`는 항상 영어로 큐레이팅되어 게시됩니다.
> - `skills_omni/`는 단방향 표면이며 기본 섭취로 다시 루프되지 않습니다.---

## 💡 Why Omni Skills

>**단지 "폴더에 기술을 갖춘 또 다른 저장소"가 아닙니다.**
> Omni Skills는 더 강력한 계약과 더 넓은 런타임 영역을 갖추고 있습니다.

| 원한다면... | 📁 일반적인 기술 저장소 | ✨ 옴니 스킬 |
|:-------------|:---------|:---------------|
| 실제 조수에 설치 | 수동 복사 또는 사용자 정의 스크립트 | `npx omni-skills`, 안내 설치, 시각적 UI, 선택적 `--skill` 및 `--bundle` |
| 기술 검색 및 비교 | 마크다운을 수동으로 찾아보기 | 카탈로그 생성, 필터링, 번들 기획, 검색, 비교, 추천 |
| 여러 도구에서 동일한 데이터 사용 | 도구별로 별도의 로직 | CLI, API, MCP 및 A2A용 공유 매니페스트 및 카탈로그 |
| MCP 클라이언트 구성 | 직접 편집한 파일 | `config-mcp`, 로컬 사이드카 미리보기, 생성된 레시피 및 허용 목록에 있는 쓰기 |
| 신뢰 릴리스 | 최선을 다해 포장 | 체크섬, 서명된 아카이브, 스캐너 확인, CI 릴리스 및 실행 전 게시 |
| 커뮤니티 섭취 관리 | 어떤 땅이든 그대로 유지됩니다 | `skills/`의 기본 섭취, 속성이 있는 `skills_omni/`의 선별된 영어 파생어 |---

## 🔌 Runtime Surfaces

Omni Skills는 단순한 기술 라이브러리가 아닙니다. 생성된 동일한 카탈로그를 사용하는**4개의 런타임 표면**을 노출합니다.

| 표면 | 상태 | 그것이 하는 일 | 예 |
|:---------|:------|:-------------|:---------|
| 🖥️**CLI**| ✅ 이용 가능 | 찾기, 설치, 진단, 시각적 UI, 부팅 서비스, 연기 검사 | `npx 만능 의사` |
| 🌐**카탈로그 API**| ✅ 이용 가능 | 읽기 전용 카탈로그, 검색, 번들, 비교, 설치 계획, 다운로드 | `npx omni-skills api --port 3333` |
| 🔌**MCP**| ✅ 이용 가능 | 검색, 권장사항, 설치 미리보기, 로컬 사이드카, 구성 흐름 | `npx omni-skills mcp 스트림 --local` |
| 🤖**A2A**| ✅ 이용 가능 | 작업 수명주기, 핸드오프, 폴링, 스트리밍, 취소, 지속성 | `npx 옴니 스킬 a2a --port 3335` |

## 🛡️ Security and Release Posture

> Omni Skills는 일반 마크다운 저장소보다 강력한 릴리스 및 검증 스토리를 제공합니다.### 🧪 Validation and Smoke Checks

```bash
npm run validate         # Skill validation and metadata generation
npm run build            # Full build pipeline
npm test                 # Automated tests
npm run smoke            # Full release preflight
```

## 🗂️ Repository Layout

| 경로 | 목적 |
|:------|:---------|
| 📂 `기술/` | 표준 저작 기술 및 기본 섭취 |
| ✨`skills_omni/` | 선별된 옴니(Omni)가 관리하는 강화 파생상품 |
| 📖`문서/` | 사용자, 기여자, 아키텍처, 운영 및 사양 문서 |
| 📦`거리/` | 생성된 매니페스트, 번들, 카탈로그 및 아카이브 |
| 📁`데이터/` | 번들 정의 및 정적 지원 데이터 |
| 🧠`패키지/카탈로그-코어/` | 공유 카탈로그 런타임 |
| 🌐 `패키지/서버-api/` | 읽기 전용 HTTP API |
| 🔌`패키지/서버-mcp/` | MCP 서버 및 로컬 사이드카 |
| 🤖 `패키지/서버-a2a/` | A2A 런타임 및 작업 조정 |
| 🖥️`도구/빈/` | CLI 진입점 |
| 📚`도구/lib/` | 설치 프로그램 및 UI 도우미 |
| ⚙️`도구/스크립트/` | 검증, 생성, 릴리스 및 테스트 스크립트 |

>**ℹ️**`dist/`는 생성된 아티팩트가 설치, API, MCP, A2A, 연기 및 릴리스 계약의 일부이기 때문에 의도적으로 버전이 지정됩니다.---
