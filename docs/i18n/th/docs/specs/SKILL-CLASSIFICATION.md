## 📊 Status

| เอาท์พุท | สร้าง |
|:-------|:----------|
| ✅ รูท `metadata.json` | สรุปทั้งพื้นที่เก็บข้อมูล |
| ✅ ต่อทักษะ `skills/<skill>/metadata.json` | การจำแนกประเภทส่วนบุคคล |
| ✅ แคตตาล็อก `dist/catalog.json` | แค็ตตาล็อกที่เผยแพร่พร้อมคะแนน |
| ✅ แสดง `dist/manifests/<skill>.json` | ข้อมูลที่เครื่องอ่านได้ต่อทักษะ |

- 32 ทักษะที่เผยแพร่

## 🎯 Purpose

ตัวแยกประเภทช่วยให้ทุกทักษะมี**โปรไฟล์ที่เครื่องอ่านได้สอดคล้องกัน**ก่อนที่จะเข้าถึงแค็ตตาล็อก ทำหน้าที่สี่งาน:

1. 📋**แยกวิเคราะห์**— YAML frontmatter และ markdown body
2. 🏷️**ทำให้เป็นมาตรฐาน**— ป้ายกำกับหมวดหมู่เป็นอนุกรมวิธานตามรูปแบบบัญญัติ
3. 📊**จัดประเภท**— ความสมบูรณ์ แนวทางปฏิบัติที่ดีที่สุด คุณภาพ และการให้คะแนนด้านความปลอดภัย
4. 📁**ปล่อย**— สิ่งประดิษฐ์ของข้อมูลเมตาที่ใช้โดยสคริปต์บิลด์ เอกสาร และ CI---

| สัญญาณ | ตรวจสอบอะไร |
|:-------|:---------------|
| 📛 คุณภาพทาก | การจัดรูปแบบฟิลด์ `ชื่อ` |
| 📝 คำอธิบาย | ความชัดเจน ความยาว ข้อมูล |
| 📐 โครงสร้าง | ส่วนเอกสารและลำดับชั้น |
| 💡 ตัวอย่าง | รหัสรั้วและบล็อกตัวอย่าง |
| 🔗 ข้อมูลอ้างอิง | เชื่อมโยง `references/`, `scripts/` และตัวช่วยแพ็คสนับสนุน |
| 🧰 การใช้งาน | ตัวอย่างสคริปต์ในเครื่องที่รันได้และข้อมูลโค้ดเวิร์กโฟลว์ที่เป็นรูปธรรม |

| 📚 ความคุ้มครอง | `เมื่อใดควรใช้`, `แนวทางปฏิบัติที่ดีที่สุด`, `การแก้ไขปัญหา` และ `แหล่งข้อมูลเพิ่มเติม` |

**ลำดับปัจจุบัน**

| ชั้น | ช่วงคะแนน |
|:-----|:-----------|
| `ยอดเยี่ยม` | 90-100 |
| `ดี` | 70-89 |
| `ยุติธรรม` | 50-69 |
| `จำเป็นต้องทำงาน` | 0-49 |

ผู้บันทึกคะแนนมีเจตนา**ความหมายเพียงพอที่จะสร้างการกระจาย**ไปสู่ทักษะที่บรรลุนิติภาวะ ทักษะที่มีโครงสร้างสะอาดสามารถทำคะแนนได้ดี แต่การจะไปถึงแถบระดับบนนั้นยังต้องมีสัญญาณเชิงลึก เช่น:

- มีหลายตัวอย่าง ไม่ใช่แค่ตัวอย่างเดียว
- กรณีการแก้ไขปัญหาหลายกรณี
- คำแนะนำทักษะที่เกี่ยวข้อง
- ชุดสนับสนุนท้องถิ่นที่สมบูรณ์ยิ่งขึ้น
- ตระกูลสนับสนุนมากกว่าหนึ่งกลุ่ม นอกเหนือจากร้อยแก้วธรรมดา โดยในอุดมคติแล้ว รวมถึง `ตัวแทน/` หรือ `ทรัพย์สิน/` ที่พวกเขาเพิ่มการใช้ซ้ำจริง
- ส่วน `## ขั้นตอนการทำงาน` โดยเฉพาะพร้อมขั้นตอนที่นับได้
- ตารางการปฏิบัติงานขนาดเล็กหรือแผนผังการตัดสินใจอย่างน้อยหนึ่งรายการ เมื่อปรับปรุงความชัดเจนในการดำเนินการ
- มีความจำเพาะในการปฏิบัติงานมากกว่าเทมเพลตธรรมดา
- ความลึกของขั้นตอนการทำงานที่ชัดเจนยิ่งขึ้นและสินทรัพย์สนับสนุนการตัดสินใจ
- ความลึกของแพ็กสนับสนุนที่มากกว่าไฟล์ `อ้างอิง/` หนึ่งไฟล์และสคริปต์ที่เชื่อมโยงหนึ่งไฟล์
- ไฟล์สนับสนุนที่สามารถนำมาใช้ซ้ำได้มากพอที่จะให้ความรู้สึกเหมือนเป็นชุดเวิร์กโฟลว์ ไม่ใช่ส่วนเสริมโน้ตเดียว
- ความหนาแน่นในการปฏิบัติงานเพียงพอที่จะแยกโครงร่างที่สวยงามออกจากชุดเวิร์กโฟลว์ที่นำกลับมาใช้ใหม่ได้

นั่นหมายความว่าทักษะที่มีโครงสร้างสมบูรณ์จะยังคงสามารถเข้าสู่ยุค 90 แทนที่จะเป็น `100` หากชุดสนับสนุนแคบลง สินทรัพย์ในการตัดสินใจบางลง หรือความหนาแน่นในการปฏิบัติงานต่ำกว่าทักษะที่แข็งแกร่งที่สุดในแค็ตตาล็อก---

### 🛡️ Security Score (0-100)

ชั้นความปลอดภัยจะรวมเอา:

| เครื่องสแกน | เปิดใช้งานเสมอ | มันทำอะไร |
|:--------|:---------------|:-------------|
| 🔍**คงที่**| ✅ ใช่ | สแกน SKILL.md ไฟล์แพ็กเกจ และสคริปต์ |
| 😢**ClamAV**| ⚙️ ไม่บังคับ | การสแกนมัลแวร์ผ่าน `clamscan` |
| 🔒**ไวรัสรวม**| ⚙️ ไม่บังคับ | ค้นหาแฮช (ไม่มีการอัปโหลด) |

**กลุ่มกฎเครื่องสแกนแบบคงที่:**
- 🎭 รูปแบบการฉีดและการกรองที่รวดเร็ว
- 💣 คำสั่งเชลล์ทำลายล้าง
- 🔑 ข้อมูลรับรองหรือเส้นทางระบบปฏิบัติการที่น่าสงสัย
- ⚠️ สคริปต์ดั้งเดิมที่มีความเสี่ยง (`shell=True`, `pickle.load`, `eval`, `extractall`)

**รูปแบบเอาต์พุตการรักษาความปลอดภัย:**```json

    "virustotal": { "enabled": false, "status": "disabled" }
  }
}
```

### 🪝 Optional Git Hooks

```bash
npm run hooks:install
```

วิธีนี้จะกำหนดค่า `git` ให้ใช้ `.githooks/pre-commit` ซึ่งจะสร้างข้อมูลเมตาและส่วนแค็ตตาล็อกใหม่ก่อนที่จะคอมมิตและจัดการไฟล์ที่สร้างขึ้นโดยอัตโนมัติ
