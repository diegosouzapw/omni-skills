## 🎯 Purpose

Классификатор придает каждому навыку**согласованный машиночитаемый профиль**, прежде чем он попадет в каталог. Он выполняет четыре работы:

1. 📋**Парсинг**— фронтматер YAML и тело уценки.
2. 🏷️**Нормализация**— метки категорий для канонической таксономии.
3. 📊**Классифицировать** – оценка зрелости, лучших практик, качества и безопасности.
4. 📁**Emit**— артефакты метаданных, используемые сценариями сборки, документацией и CI.---

| Уровень | Этикетка | Критерии |

### 📋 Best Practices Score (0-100)

Эвристика оценивает:

| Сигнал | Что он проверяет |
|:-------|:---------------|
| 📛 Качество слизней | форматирование поля `имя` |
| 📝 Описание | Ясность, длина, информативность |
| 📐 Структура | Разделы документа и иерархия |
| 💡 Примеры | Ограждения кода и блоки примеров |
| 🔗 Рекомендации | Связанные локальные `references/`, `scripts/` и помощники пакетов поддержки |
| 🧰 Работоспособность | Примеры запускаемых локальных сценариев и конкретные фрагменты рабочих процессов |
| 🧩 Глубина пакета поддержки | Несколько семейств поддержки, файлы многократного использования, метаданные агентов и операционные активы |
| 🩺 Устранение неполадок | Явные пары «Симптомы» и «Решение» |
| 📚 Покрытие | Разделы «Когда использовать», «Рекомендации», «Устранение неполадок» и «Дополнительные ресурсы» |
| 🌐 Портативность | Формулировка, не зависящая от инструмента |
| 📅 Свежесть | Избегание жестко закодированных дат |

**Текущий уровень**

| Уровень | Диапазон очков |
|:-----|:-----------|
| `отлично` | 90-100 |
| `хорошо` | 70-89 |
| `справедливый` | 50-69 |
| `нужна работа` | 0-49 |

Секретарь намеренно**достаточно семантичен, чтобы обеспечить разброс**по зрелым навыкам. Навык с чистой структурой может получить хорошие оценки, но для достижения верхней границы ему также необходимы сигналы глубины, такие как:

- несколько примеров, а не один
- несколько случаев устранения неполадок
- руководство по соответствующим навыкам
- более богатые локальные пакеты поддержки
- более одного семейства поддержки, выходящее за рамки простой прозы, в идеале включая `агенты/` или `активы/`, где они добавляют реальное повторное использование
- специальный раздел `## Workflow` с подсчетом шагов.
- хотя бы одну небольшую операционную таблицу или карту решений, если это повышает ясность выполнения
- больше оперативной специфики, чем у простого шаблона
- более четкая глубина рабочего процесса и ресурсы поддержки принятия решений
- глубина пакета поддержки выходит за рамки одного файла `references/` и одного связанного скрипта.
- достаточно многоразовых вспомогательных файлов, чтобы они выглядели как комплект рабочего процесса, а не надстройка для одной заметки.
- достаточная операционная плотность, чтобы отделить отшлифованный контур от многоразового рабочего комплекта

Это означает, что структурно завершенный навык все равно может оказаться на уровне 90 вместо «100», если его пакет поддержки уже, его ресурсы для принятия решений тоньше или его операционная плотность ниже, чем у самых сильных навыков в каталоге.---

### 🛡️ Security Score (0-100)

Уровень безопасности сочетает в себе:

| Сканер | Всегда включено | Что он делает |
|:--------|:---------------|:-------------|
| 🔍**Статический**| ✅ Да | Сканирует SKILL.md, упакованные файлы и скрипты |
| 🦠**КлэмАВ**| ⚙️ Необязательно | Сканирование вредоносных программ с помощью clamscan |
| 🔒**VirusTotal**| ⚙️ Необязательно | Поиск по хешу (без загрузки) |

**Семейства правил статического сканера:**
- 🎭 Быстрые схемы инъекций и эксфильтрации
- 💣 Деструктивные команды оболочки
- 🔑 Подозрительные учетные данные или пути к ОС.
- ⚠️ Рискованные примитивы скриптов (`shell=True`, `pickle.load`, `eval`, `extractall`)

## ⚙️ Workflow Integration

```bash
npm run validate              # Validate + regenerate metadata
npm run build                 # Full build with catalog + archives
npm run taxonomy:report       # Show category drift suggestions
```

### 🪝 Optional Git Hooks

```bash
npm run hooks:install
```

Это настраивает git на использование .githooks/pre-commit, который регенерирует метаданные и артефакты каталога перед фиксацией и автоматически размещает сгенерированные файлы.
