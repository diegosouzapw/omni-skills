>**Поведенческий контракт для пользовательского интерфейса терминала на основе Ink, предоставляемого `omni-skills ui`.**---

## 1. Scope

Визуальная оболочка — это управляемая поверхность продукта поверх существующего интерфейса командной строки и механизма установки.

Он не заменяет:

- экспертное использование CLI на основе флагов
- `инструменты/bin/install.js`
- порядок установки текста с подсказками
- Поведение API, MCP или A2A во время выполнения.

Он определяет:

- поведение `omni-skills ui`
— запасной контракт для `omni-skills ui --text`
- локальное состояние и постоянство предустановок
- управляемый предварительный просмотр запуска сервиса
- повторяемость для недавних установок и сервисных запусков---

## 2. Entry Rules

### 2.1 Visual Mode

`omni-skills ui` запускает визуальную оболочку на основе Ink.

Визуальная оболочка — это основной интерфейс терминала для неспециалистов, позволяющий:

- установить потоки
- обнаружение и установка по каталогу
- запуск МКП
- запуск API
- Стартап А2А
- передача врача и курения### 2.2 Text Fallback

`omni-skills ui --text` запускает резервный интерфейс на основе строки чтения.

Это остается полезным, когда:

- терминал не может правильно отображать более богатую оболочку
- поведение в необработанном режиме ограничено
- предпочтителен минимальный резервный текст### 2.3 Handoff Rule

Визуальная оболочка не переопределяет среду выполнения службы или запись установки напрямую.

После предварительного просмотра и подтверждения он завершает работу и передает выполнение существующей точке входа CLI с эквивалентными аргументами и переменными среды.---

## 3. Home Screen Contract

На главном экране должно быть показано:

- установить навыки
- найти и установить
- повторять недавние установки, если они есть
- запускать сохраненные настройки установки, если они присутствуют
- запустить службу
- повторять недавние услуги, если они присутствуют
- запускать сохраненные сервисные пресеты, если они есть
- доктор
- курить
- выход

Также должен появиться главный экран:

- текущая доступность опубликованного пакета
- местный штат учитывает недавние, предустановки и избранное---

## 4. Install Flow Contract

Процесс установки визуальной оболочки должен поддерживать:

- известный целевой выбор клиента
- выбор индивидуального пути
- полная установка библиотеки
- установка одним навыком
- установка одним пакетом
- поиск, затем установка
- предварительный просмотр перед записью
- сохранение предустановок
- любимый навык или переключение пакета

Предварительный просмотр должен показывать:

- решена целевая метка
- решенный путь
- установить прицел
- выбранный навык или пакет, если применимо
- эквивалентная команда CLI---

## 5. Service Flow Contract

Визуальная оболочка должна направлять запуск для:### 5.1 MCP

- транспорт: `stdio`, `stream`, `sse`
- режим: «только для чтения» или «локальный»
- конфигурация хоста/порта для сетевого транспорта
- явный предварительный просмотр команды### 5.2 API

- хозяин
- порт
- базовый или усиленный профиль
- усиленный носитель или аутентификация по ключу API
- усиленные параметры ограничения скорости
- включение журнала аудита
- явный предварительный просмотр команды### 5.3 A2A

- хозяин
- порт
- тип хранилища: `memory`, `json`, `sqlite`
- путь к сохранению для устойчивых режимов
- исполнитель: `inline`, `process`
- режим SQLite с поддержкой очереди
- интервал опроса и продолжительность аренды для режима общей аренды
- явный предварительный просмотр команды---

## 6. Local State Contract

Визуальная оболочка сохраняет локальное состояние в:```text
~/.omni-skills/state/ui-state.json
```

В состав штата в настоящее время входят:

- недавние установки
- недавние запуски услуг
- именованные пресеты установки
- именованные сервисные пресеты
- любимые навыки
- любимые наборы

Оболочка должна поддерживать:

- воспроизведение последних установок
- воспроизведение последних запусков сервиса
- повторное использование именованных пресетов установки
- повторное использование пресетов именованных сервисов---

## 7. Compatibility Contract

Визуальная оболочка аддитивна.

Эти потоки должны оставаться действительными и стабильными:

- `npx omni-skills --cursor --skill omni-figma`
- `npx omni-skills --bundle devops`
- `npx omni-skills install --guided`
- `npx omni-skills find Figma --tool курсор --install --yes`
- `npx omni-skills mcp поток --local`
- `npx omni-skills api --port 3333`
- `npx omni-skills a2a --port 3335`

Визуальная оболочка никогда не должна навязывать себе явные пути экспертных команд.---

## 8. Safety Contract

Визуальная оболочка должна явно указывать состояние и запись.

Оно должно:

- предварительная установка устанавливается перед передачей записи
- предварительный просмотр команд запуска сервиса перед выполнением
- храните секретный материал вне предварительного просмотра команд в виде открытого текста, где это возможно.
- сохранять состояние только локально
- сохранить неинтерактивное поведение CLI вне визуальной оболочки
