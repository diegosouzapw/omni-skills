## Szybki start

>**Szukałeś umiejętności kodowania AI, umiejętności Claude Code, umiejętności kursora, umiejętności Codex CLI, umiejętności Gemini CLI, umiejętności antygrawitacyjnych lub instalowalnych bibliotek `SKILL.md`?**
> Jesteś we właściwym miejscu.### 1️⃣ What is this?

Omni Skills to**instalowalny katalog umiejętności i środowisko wykonawcze**dla asystentów kodowania AI. W swej istocie jest to publiczne repozytorium podręczników `SKILL.md` wielokrotnego użytku — ale w przeciwieństwie do zwykłych kolekcji umiejętności, repozytorium**jest**warstwą dystrybucji i środowiska wykonawczego.

### 2️⃣ Quick Start

```bash
# Install with guided flow
npx omni-skills

# Or install directly for Antigravity (default outside TTY)
npx omni-skills install --guided
```

### 3️⃣ Verify

```bash
test -d ~/.gemini/antigravity/skills && echo "✅ Skills installed"
```

## 🧩 Core Concepts

Przed porównaniem pakietów lub wybraniem ścieżki instalacji pomocne jest zrozumienie tych pięciu elementów:

| Koncepcja | Co to znaczy |
|:------------|:------------|
| 🧠**Umiejętności**| Podręczniki `SKILL.md` wielokrotnego użytku, które uczą asystenta, jak dobrze wykonać przepływ pracy |
| 📦**Artefakty katalogu**| Wygenerowane dane wyjściowe w formacie JSON i archiwum umożliwiające wyszukiwanie, porównywanie, pobieranie i instalowanie |
| 🔌**Konfiguracja MCP**| Konfiguracja po stronie klienta dla asystentów w celu odkrycia umiejętności Omni za pomocą narzędzi MCP |
| 🤖**Środowisko wykonawcze A2A**| Orkiestracja między agentami w celu wykrywania, rekomendowania i przekazywania planu instalacji |
| ✨**Wybrane wyniki**| `skills_omni/` — udoskonalona powierzchnia utrzymywana przez Omni, oddzielona od natywnego wlotu w górę rzeki |

>**📝 Zasady natywne/wyselekcjonowane:**
> - `umiejętności/` akceptuje natywne wykorzystanie materiałów źródłowych w dowolnym języku
> - `skills_omni/` jest zawsze przygotowywane i publikowane w języku angielskim
> - `skills_omni/` jest powierzchnią jednokierunkową i nie wraca do natywnego spożycia---

## 💡 Why Omni Skills

>**Nie tylko „kolejne repozytorium z umiejętnościami w folderach.”**
> Omni Skills ma silniejszy kontrakt i szerszy obszar działania.

| Jeśli chcesz… | 📁 Repozytorium typowych umiejętności | ✨Umiejętności Omni |
|:-------------|:----------------------|:-------------|
| Zainstaluj w prawdziwym asystencie | Kopia ręczna lub skrypt niestandardowy | `npx omni-skills`, instalacja z przewodnikiem, wizualny interfejs użytkownika, selektywne `--skill` i `--bundle` |
| Wyszukaj i porównaj umiejętności | Przeglądaj przecenę ręcznie | Wygenerowany katalog, filtrowanie, planowanie pakietów, wyszukiwanie, porównywanie i rekomendowanie |
| Używaj tych samych danych w różnych narzędziach | Oddzielna logika dla każdego narzędzia | Udostępnione manifesty i katalog dla CLI, API, MCP i A2A |
| Skonfiguruj klientów MCP | Ręczna edycja plików | `config-mcp`, lokalne podglądy sidecar, wygenerowane przepisy i zapisy na liście dozwolonych |
| Zaufanie uwalnia | Opakowanie typu best-effort | Sumy kontrolne, podpisane archiwa, weryfikacja skanera, wydanie CI i opublikowanie wstępnej inspekcji |
| Zarządzaj spożyciem przez społeczność | Cokolwiek wyląduje, pozostanie takie, jakie jest | Natywne wykorzystanie w `skills/`, wyselekcjonowane angielskie pochodne w `skills_omni/` z przypisaniem |---

## 🔌 Runtime Surfaces

Omni Skills to nie tylko biblioteka umiejętności. Udostępnia**cztery powierzchnie wykonawcze**, które korzystają z tego samego wygenerowanego katalogu:

| Powierzchnia | stan | Co to robi | Przykład |
|:------------|:------|:------------|:------------|
| 🖥️**CLI**| ✅Dostępne | Znajdź, zainstaluj, zdiagnozuj, wizualny interfejs użytkownika, usługi rozruchowe, kontrole dymu | `npx lekarz omni-umiejętności` |
| 🌐**API katalogu**| ✅Dostępne | Katalog tylko do odczytu, wyszukiwanie, pakiety, porównywanie, instalowanie planów, pobieranie | `npx omni-skills api --port 3333` |
| 🔌**MCP**| ✅Dostępne | Wykrywanie, rekomendacja, podgląd instalacji, lokalny wózek, przepływy konfiguracji | `npx omni-skills mcp strumień --local` |
| 🤖**A2A**| ✅Dostępne | Cykl życia zadania, przekazanie, odpytywanie, przesyłanie strumieniowe, anulowanie, trwałość | `npx omni-skills a2a --port 3335` |

## 🗂️ Repository Layout

| Ścieżka | Cel |
|:---------|:------------|
| 📂 `umiejętności/` | Kanoniczne umiejętności autorskie i natywny spożycie |
| ✨ `umiejętności_omni/` | Wyselekcjonowane, ulepszone instrumenty pochodne objęte wszechstronną obsługą |
| 📖 `dokumenty/` | Użytkownik, współautor, architektura, operacje i dokumentacja specyfikacji |
| 📦 `odległość/` | Wygenerowane manifesty, pakiety, katalog i archiwa |
| 📁 `dane/` | Definicje pakietów i statyczne dane pomocnicze |
| 🧠 `pakiety/rdzeń-katalogu/` | Środowisko wykonawcze katalogu udostępnionego |
| 🌐 `pakiety/serwer-api/` | Interfejs API HTTP tylko do odczytu |
| 🔌 `pakiety/serwer-mcp/` | Serwer MCP i lokalny wózek |
| 🤖 `pakiety/serwer-a2a/` | Koordynacja środowiska wykonawczego i zadań A2A |
| 🖥️ `narzędzia/kosz/` | Punkty wejścia CLI |
| 📚 `narzędzia/lib/` | Instalator i pomocnicy interfejsu użytkownika |
| ⚙️ `narzędzia/skrypty/` | Walidacja, generowanie, wydawanie i testowanie skryptów |

>**ℹ️**`dist/` ma celowo wersjonowaną wersję, ponieważ wygenerowane artefakty są częścią umowy dotyczącej instalacji, API, MCP, A2A, dymu i wydania.---
